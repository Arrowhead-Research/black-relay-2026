What it is

A technical systems-integration event, not a generic hackathon.

Participants work remotely over roughly four months to build partially fieldable subsystems with documentation, interfaces, and test evidence. The program culminates in a live in-person integration and deployment phase.

The final system should be deployable in an unfamiliar building with minimal setup and continue functioning when connectivity or individual components fail.

The challenge

Rapidly emplace a portable safehouse system under realistic constraints.

How can a small team provide local awareness, detect compromise or degradation, preserve essential functions, and support operator decisions during a multi-day stress period?

Technical lanes

Four teams, one integrated safehouse stack.

Fabrication

Make the system physically fieldable, modular, transportable, serviceable, and fast to install.

  • Collaborative CAD
  • Enclosures and mounts
  • Cable routing
  • Setup and teardown workflows

Detection

Turn raw sensor inputs into meaningful site awareness with confidence and corroboration.

  • Sensor architecture
  • Ingest pipelines
  • Event schema
  • Alert prioritization

Continuity

Keep essential functions alive and visible when links, components, or power degrade.

  • Power model
  • Edge compute
  • Backhaul progression
  • Health telemetry

TAK

Transform technical outputs into clear operator workflows, overlays, and decisions.

  • TAK plugin concept
  • Common ingest contract
  • Alert hierarchy
  • Health visualization

Build rhythm

Four remote months, then live deployment and stress testing.

  1. Month 1 — Requirements and ArchitectureDefine subsystem concepts, interfaces, assumptions, and integration points.
  2. Month 2 — PrototypingBuild early subsystem prototypes and validate technical feasibility.
  3. Month 3 — Cross-Team IntegrationConnect subsystems, refine interfaces, discover failure modes, and iterate.
  4. Month 4 — Hardening and RehearsalImprove reliability, documentation, installation workflow, and demo readiness.
  5. In-Person Event — Deployment and Stress TestingInstall, operate, and evaluate the integrated safehouse stack in a live setting.

Expected outputs

Arrive with more than a prototype.

Each lane should produce a credible, explainable package that can be integrated, tested, and operated.

  • Subsystem concept and architecture diagram
  • Interface definition for consumed and published data
  • BOM or dependency list
  • Installation or deployment notes
  • Test plan, evidence, risks, and open issues
  • Final in-person demo package

Who should apply

Builders with useful starting competence.

Black Relay is designed for military technologists, engineers, RF and communications practitioners, hardware builders, CAD contributors, cybersecurity specialists, TAK developers, and edge compute or sensor systems builders.

You do not need to know everything on day one, but you should bring hands-on skill in at least one relevant domain. Coaching is available; starting from scratch is not the goal.

Success criteria

Credible. Modular. Survivable. Operator-usable.

Installs quickly in unfamiliar structures Continues under degraded conditions Detects tampering or failure Preserves critical information Presents useful decisions to operators Uses clear interfaces and documentation
